# 公益机器人视频数据自动化采集与清洗流水线 (Project Blueprint)

## 一、 项目全局目标与核心约束

* **最终目标：** 2026年3月底前，交付 4000 小时高质量人体动作视频训练数据。
* **交付节奏：** 允许分批次交付（目标：每周约 1000 小时）。
* **预算限制：** 约 10,000 元人民币。
* **视频质量硬性标尺（The Golden Rules）：**
1. **时长与连贯性：** 单个片段 5-20s，无剪辑/运镜。
2. **画质要求：** 分辨率严格 ≥ 720p。
3. **主体数量：** 画面中仅有且必须只有**单人**。
4. **机位要求：** 摄像机绝对静止（固定机位）。
5. **空间完整度：** 人体解剖结构完整，**脚部与地面接触必须清晰可见**。
6. **主体占比：** 横屏视频人高 > 画面高度 1/2；竖屏视频人高 > 画面高度 1/3。



## 二、 核心技术栈与工具链

放弃传统的本地单机死磕，采用**云原生分布式高并发架构**。

* **开发与控制环境：** 本电脑为linux ubuntu系统笔记本电脑，配置有限。
* **调度中心 (Master)：** AWS EC2 (t3.medium) + FastAPI + PostgreSQL（基于行级锁 `FOR UPDATE SKIP LOCKED` 实现健壮的任务队列）。
* **算力节点 (Worker)：** 国内算力租赁平台（如 AutoDL、揽睿星舟）的 RTX 3090/4090 竞价实例 + Docker。
* **网络与反爬：** 顶级动态住宅 IP 代理池（BrightData / Oxylabs）配合 `yt-dlp`。
* **物理切割：** `PySceneDetect`（基于像素差异的高速场景切换检测）。
* **视觉核心引擎：** **YOLO-Pose** (YOLOv8/v11)（全面替代 MediaPipe，原生支持多人排除、精确 Bounding Box 计算占比、以及高置信度脚踝关键点检测）。
* **语义打标大脑：** 豆包大模型 (Doubao-Seed-2.0-mini) API（处理多图抽帧推理，成本极低）。
* **数据交付中心：** AWS S3 / 阿里云 OSS。

## 三、 自动化工作流漏斗 (The Data Funnel)

整个 Pipeline 部署在 Worker 节点上，遵循“尽早丢弃、极力压榨算力”的原则：

1. **领任务与高速拉取：** Worker 向 Master 请求关键词/URL，通过代理池使用 `yt-dlp` 下载原始视频（直接过滤掉 <720p 的源）。
2. **免算力物理切割：** 使用 `PySceneDetect` 快速遍历视频流，识别并剔除所有运镜，硬切出 5-20s 的“纯净片段”。
3. **极简抽帧与 YOLO-Pose 过滤（核心壁垒）：**
* 不逐帧推理。从每个片段抽取 3 帧（首、中、尾）。
* **过检条件 A（单人）：** YOLO 返回的 `len(boxes) == 1`。
* **过检条件 B（占比）：** `box_height / frame_height > 0.5` (或 0.33)。
* **过检条件 C（触地）：** 脚踝关键点 (Ankles) 置信度 score > 0.7 且处于画面底部。
* *只要不满足，直接 `os.remove()` 物理删除该片段。*


4. **低成本 VLM 语义分类：** 将幸存片段的关键帧拼接为宫格图，发送至豆包大模型 API 进行最终类别打标（如：常规行走、武术、日常抓取）。
5. **上云与状态同步：** 合格片段直传 AWS S3，Worker 向 PostgreSQL 更新进度，随后清空本地磁盘缓存，进入下一个循环。

## 四、 阶段性战术与执行计划

### 阶段一：冷启动与“白嫖”策略 (本周首要任务)

利用开源学术数据集度过第一周的 1000 小时交付期，为搭建云端流水线争取时间。

* **目标源：** **NTU RGB+D 120**、**Kinetics-700**、**PKU-MMD**。
* **动作：** 编写脚本批量下载并解析这些已打标的高质量视频（天然满足固定机位要求），通过极简的 OpenCV 脚本剔除分辨率不达标的部分。

### 阶段二：云端架构搭建 (第 2-4 天)

* 部署 AWS EC2 上的 FastAPI 调度中心与 PostgreSQL 数据库。
* 在本地 WSL2 环境下完成 Worker 节点核心 Python 代码的编写与 Docker 镜像打包。
* 跑通单节点测试：下载 -> 切片 -> YOLO 抽帧 -> 豆包分类 -> 上传 S3。

### 阶段三：兵团作战与监控 (第 5 天至月底)

* 在算力平台批量拉起数十台 RTX 3090/4090 实例，注入住宅 IP 代理池。
* 编写本地 Streamlit 监控看板，实时查看各类别完成度与系统吞吐量。

## 五、 预算控制分配 (约 10,000 元)

1. **云端 GPU 算力 (约 4,000 - 5,000 元)：** 租赁多台 RTX 3090/4090 节点并发处理。
2. **存储与交付网络费 (约 2,000 - 3,000 元)：** S3/OSS 存储费及向甲方交付时的下行流量费（若甲方允许，可购买移动硬盘邮寄以大幅节省此费用）。
3. **代理池费用 (约 1,000 - 1,500 元)：** 购买高质量住宅 IP 流量，确保爬虫 24 小时存活。
4. **API 与调度成本 (约 500 元)：** 豆包大模型多模态调用费（极低）及 EC2 基础服务器费用。

## 六、 需求方会议纪要准备 (Talk Track)

在接下来的沟通中，确立技术合伙人形象，管理预期：

1. **提出开源数据引用的决议：** 提议在前期合法引入 NTU RGB+D 等学术界黄金标准数据集，迅速覆盖高危/冷门类别（如跌倒保护），确保系统 Baseline 质量。

* 确认人物背对镜头但满足解剖结构完整的片段是否有效。
* 确认非遮挡性平台水印是否可接受。


3. **工程客观规律同步：** 阐明自然网络视频中“满足硬性6条规则”的出材率极低（约 5%），展现你通过云原生高并发流水线解决此问题的专业性。
